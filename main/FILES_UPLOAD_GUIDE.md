# Система загрузки и управления файлами

## Описание

Система позволяет преподавателям загружать файлы двух типов:
- **УМК (Учебно-методический комплекс)** - PDF, DOC, DOCX, PPT, PPTX
- **ПК (Документы о повышении квалификации)** - PDF, JPG, PNG

## Архитектура

### Бэкенд (Node.js + Express)
- **Маршрут**: `POST /files/upload` - загрузка файлов
- **Маршрут**: `GET /files` - получение списка файлов
- **Маршрут**: `GET /files/:id` - информация о файле
- **Маршрут**: `DELETE /files/:id` - удаление файла
- **Хранилище**: `/uploads/*` - директория с загруженными файлами
- **БД**: Таблица `files` в PostgreSQL

### Фронтенд (React + TypeScript)
- **Сервис**: `src/services/files.ts` - API операции
- **Компонент**: `src/components/UploadUMK.tsx` - загрузка УМК и КП
- **Компонент**: `src/components/UploadPK.tsx` - загрузка документов ПК
- **Страница**: `src/pages/UploadUMKPage.tsx` - страница УМК
- **Страница**: `src/pages/UploadPKPage.tsx` - страница ПК

## Функциональность

### Загрузка файлов
1. Перетащите файлы в область загрузки или нажмите для выбора
2. Выберите одного или несколько файлов
3. Нажмите кнопку "Загрузить"
4. Файлы будут загружены на сервер и сохранены в БД

### Просмотр загруженных файлов
- Список загруженных файлов отображается под формой загрузки
- Каждый файл показывает название, тип и дату загрузки
- Файлы отсортированы по дате загрузки (новые сверху)

### Удаление файлов
- API поддерживает удаление файлов через `DELETE /files/:id`
- Физический файл и запись в БД удаляются одновременно

## API Примеры

### Загрузка файлов (JSON)
```bash
POST /files/upload
Content-Type: application/json
Authorization: Bearer <token>

{
  "fileType": "umk",
  "files": [
    {
      "name": "document.pdf",
      "content": "base64_encoded_file_content"
    }
  ]
}

Response:
{
  "success": true,
  "data": {
    "count": 1,
    "files": [
      {
        "id": 1,
        "name": "document.pdf",
        "path": "/uploads/1234567890_document.pdf",
        "uploadedAt": "2026-02-28T10:30:00.000Z"
      }
    ]
  }
}
```

### Получение списка файлов
```bash
GET /files
Authorization: Bearer <token>

Response:
{
  "success": true,
  "data": [
    {
      "id": 1,
      "name": "document.pdf",
      "type": "УМК",
      "status": "uploaded",
      "uploadedAt": "2026-02-28T10:30:00.000Z"
    }
  ]
}
```

### Удаление файла
```bash
DELETE /files/:id
Authorization: Bearer <token>

Response:
{
  "success": true
}
```

## Безопасность
- Все операции требуют авторизации (Bearer token)
- Файлы принадлежат пользователю, который их загрузил
- Пользователь не может получить доступ к файлам других пользователей
- Имена файлов санитизируются при сохранении

## Расширение функциональности

### Добавление новых типов файлов
1. Обновить `uploadFiles()` функцию в `src/services/files.ts`
2. Обновить тип `fileType` параметр
3. Обновить форму загрузки в компонентах

### Добавление новых расширений файлов
1. Обновить `accept` атрибут в `<input type="file" />`
2. Обновить `getMimeType()` функцию в `server/routes/files.ts`
3. Обновить текст подсказки в компонентах

### Добавление предпросмотра файлов
1. Создать новый маршрут `GET /files/:id/preview`
2. Добавить компонент Preview
3. Интегрировать в UploadUMK/UploadPK компоненты
