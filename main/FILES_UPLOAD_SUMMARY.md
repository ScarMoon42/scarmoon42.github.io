# Резюме: Система загрузки файлов

## Что было добавлено

### Бэкенд (Node.js/Express)

1. **Новый маршрут файлов** (`server/routes/files.ts`):
   - `POST /files/upload` - загрузка файлов с сохранением в БД и файловую систему
   - `GET /files` - список файлов текущего пользователя
   - `GET /files/:id` - информация о конкретном файле
   - `DELETE /files/:id` - удаление файла

2. **Обновлен главный сервер** (`server/index.ts`):
   - Добавлен импорт маршрута файлов
   - Регистрация маршрута `/files`
   - Статический сервис для доступа к загруженным файлам (`/uploads/*`)
   - Увеличен лимит размера JSON тела запроса до 50MB

3. **Хранение файлов**:
   - Физическое хранилище: `/uploads/` директория
   - БД хранилище: таблица `files` (уже существует в schema.prisma)

### Фронтенд (React/TypeScript)

1. **Сервис файлов** (`src/services/files.ts`):
   - `uploadFiles(fileType, files)` - загрузка файлов
   - `getFiles()` - получение списка файлов
   - `deleteFile(fileId)` - удаление файла
   - Конвертация файлов в base64 для передачи через JSON

2. **Обновлены компоненты загрузки**:
   - `src/components/UploadUMK.tsx` - полностью функциональный компонент с:
     - Drag-and-drop поддержкой
     - Выбором файлов через input
     - Валидацией типов файлов
     - Отображением загруженных файлов
     - Обработкой ошибок и успехов
   
   - `src/components/UploadPK.tsx` - аналогичный компонент для подтверждения квалификации
   
   - Страницы уже используют эти компоненты:
     - `src/pages/UploadUMKPage.tsx`
     - `src/pages/UploadPKPage.tsx`

## API Примеры

### Загрузка файлов
```bash
POST /files/upload
Content-Type: application/json
Authorization: Bearer <token>

{
  "fileType": "umk",
  "files": [
    {
      "name": "document.pdf",
      "content": "base64_encoded_content..."
    }
  ]
}
```

### Получение списка файлов
```bash
GET /files
Authorization: Bearer <token>
```

### Удаление файла
```bash
DELETE /files/1
Authorization: Bearer <token>
```

## Развертывание

Чтобы система работала, убедитесь:

1. **PostgreSQL запущена** с базой данных из миграций
2. **Redis запущена** (опционально, для сессий)
3. **Директория `/uploads` создана** или будет создана автоматически
4. **Переменные окружения установлены** (DATABASE_URL, REDIS_URL, PORT, CORS_ORIGIN)

## Тестирование

1. Запустите БД: `docker-compose up -d`
2. Запустите бэкенд: `npm run dev:server`
3. Запустите фронтенд: `npm run dev`
4. Авторизуйтесь как преподаватель
5. Перейдите на страницу загрузки УМК или ПК
6. Загрузите файл через drag-and-drop или выбор
